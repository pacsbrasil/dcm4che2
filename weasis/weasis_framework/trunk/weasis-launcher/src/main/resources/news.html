Release Notes - Weasis - Version 1.1.0

Bug

  • [WEA-40] - Cannot load weasis-i18n with dcm4chee-web3
  • [WEA-45] - Accept empty values for IDs in weasis-pacs-connector
  • [WEA-50] - Keystrokes for Window/Level (W), Measure (M) and Context Menu (Q) does not work
  • [WEA-53] - Cannot resume downloading series after changing priority
  • [WEA-56] - Only one series is removed when several are selected
  • [WEA-57] - Series progress bar is not always refresh
  • [WEA-58] - Weasis Portable hangs when loading images from the directory defined by the property "weasis.portable.dicom.directory"
  • [WEA-59] - Weasis Portable does not display split series when loading images of the CD-ROM folder
  • [WEA-60] - Some DICOM files are not deleted of the temp directory when closing the application
  • [WEA-61] - Error of buiding the views of preferences (OSGI service)

Improvement

  • [WEA-41] - Overriding jnlp template of weasis-pacs-connector
  • [WEA-42] - Add Accession Number as a request ID
  • [WEA-44] - Limit access for weasis-pacs-connector
  • [WEA-46] - weasis-pacs-connector and weasis can be on different servers
  • [WEA-47] - Upgrade to Felix framework 3.2.1
  • [WEA-48] - Supports WADO xml references file embedded in Java Web Start (jnlp)
  • [WEA-49] - From Weasis 1.1.0 the internationalized package (i18n) can be applied to any version
  • [WEA-54] - Shows indeterminate progress bar for large unique file (video, multiframe...) of a series
  • [WEA-55] - Images with non-square pixels are rectified and displayed as square pixels
  • [WEA-63] - Resize the views of layouts
  • [WEA-69] - Contextual menu for drawings (when selected)

New Feature

  • [WEA-43] - Manage in properties which IDs are allowed and which combination is necessary
  • [WEA-51] - Add series to the layout (change the layout if it cannot contain all the series)
  • [WEA-52] - New measurements tools (three points circle, polyline, perpendicular, parallel, open anlge, four points angle, Cobb's angle) and more display properties
  • [WEA-62] - Cut, copy and paste drawings into another view
  • [WEA-64] - Computing SUV for PET images
  • [WEA-65] - Anonymize DICOM annotations in display
  • [WEA-66] - Add in xsd schema httpTag (add tags in http request) and DirectDownloadFile (direct URL, not WADO)
  • [WEA-67] - Export the selected view to clipboard (with all annotations in anonymized mode)
  • [WEA-68] - New command for loading a DICOM from an URL (dicom:get -r URL)
