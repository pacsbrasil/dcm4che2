Study Metadata Service

Overview

C-FIND metadata service that will return a patient root hierarchy of results in a specified text format. Constraints to the C-FIND are applied by adding parameters that correspond to the DICOM tag names. As with any C-FIND query, constraints on the containing types can be applied as well as the requested level.

An empty result set will be returned if query constraints are not provided.

Parameters

Parameter Description Notes
PatientName Constrain by patient name DICOM formatted
AccessionNumber Constrain by the accession number of the study
studyUID Constrain by the UID of the study
ModalitiesInStudy Constrain by the modalities in the study
PatientID Constrain by the patient ID
StudyDescription Constrain by the study description Allows wildcards
contentType Determines how the metadata will be formatted:
  • text/xml Formats the metadata as an XML document according to the study.xsd schema
  • text/javascript Formats the metadata as a JSON document which groups sub elements in a 'children' property
Defaults to text/xml
help Whether to show this help message. Defaults to false
