Patient XML Service

Overview

C-FIND metadata service that will return a patient root hierarchy of results in a specified text format. Constraints to the C-FIND are applied by adding parameters that correspond to the DICOM tag names. As with any C-FIND query, constraints on the containing types can be applied as well as the requested level.

An empty result set will be returned if query constraints are not provided.

Parameters

Parameter Description Notes
PatientName Constrain by patient name. By default, this is case-insensitive, and has some rules for searching multiple names:

Single name given: searches only for an exact match on the last name

Name in LastName, FirstName format: searches for the last name and wild cards the first name.

Name in FirstName LastName format (note the missing comma to differentiate the types): searches for lastname, firstName with a wild-card.

Name in DICOM format: searches only for the exact string given

Name in quotes: removes the quotes and searches for the given string.

 DICOM formatted
PatientID Constrain by the patient ID. May automatically add a prefix, suffix if configured, as well as automatically upper or lower caseing the text.

Adds a wild-card if the patient id contains at least 4 characters.

Multiple entries can be searched for at once by separating them with a back-slash character (\). This requires exact-matches on the ID without any wild-cards. Do NOT include any wild cards in any of the multi-entry items as this will cause all patients to be returned.
PatientBirthDate Search on the patient's birth date. See Common for format.
Sex Search on the patient gender M, F or O are the allowed values.
