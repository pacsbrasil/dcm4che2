Query Parameter: regroup

The 'regroup' flag is used to determine if the series will be grouped/sorted by the server before the metadata is returned to the client. By default the flag is turned on, and to turn it off the following expression should be added to the URL query string: 'regroup=false'.

To request a study without regrouping the series the following request should be used:
http://host/wado2/?requestType=IMAGE&AccessionNumber=1234&regroup=false

NOTE: Activating regrouping of images can require the retrieval of the full DICOM header for objects being processed. This can lead to performance degradation for studies will a large number of images.

Regrouping of MG Studies

MG regrouping is done based on the laterality and orientation of each image found in the study. The laterality and orientation are determined from the Image Laterality and View Code Sequence DICOM elements.

The following steps will be performance in the grouping of MG images:

  • Images that are not oriented left in the anterior view will be flipped to this orientation.
  • Standardized series descriptions will be generated i.e. "L CC" or "R MLO"
  • Images will be grouped into the appropriate series
  • Series will sorted so that CC images are always after MLO.

Regrouping of Multiframe Studies

Multiframe studies are regrouped by the Instance Number element of the DICOM header. This will generally result in the separation of multiframes into separate series.

Regrouping of MR Studies

Not yet implemented due to a lack of suitable DICOM data.
