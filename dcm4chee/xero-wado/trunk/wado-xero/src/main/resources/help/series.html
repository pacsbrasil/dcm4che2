Series Metadata Service

Overview

C-FIND metadata service that will return a patient root hierarchy of results in a specified text format. Constraints to the C-FIND are applied by adding parameters that correspond to the DICOM tag names. As with any C-FIND query, constraints on the containing types can be applied as well as the requested level.

An empty result set will be returned if query constraints are not provided.

Parameters

Parameter Description Notes
Modality Constrain by the modality of the series
BodyPartExamined Constrain by the body part examined in the series
PPSStartDateTime Performed Procedure Step start date time.
seriesUID Constrain by UID of the series May contain some amount of text before a colon - this indicates a synthetic series. The ACTUAL series UID will be listed at the next level down, and that is the series UID to use when making most requests.
SeriesNumber Constrain by the number assigned to the series
SeriesDescription Constrain by the content of the series description Allows wildcards

Return Attributes

XML Attribute Description Notes
StationName
ManufacturerModelName
Manufacturer
* Attributes from image level Attributes from the image level maybe included at the series level in order to decrease redundancy. For instance, the rows/columns maybe the same for 1000 or more images, as are min/max pixel, so these can be included at the series level instead.

Return Child Elements

XML Element Description Notes
image Indicates that there is an image child of this object
report Indicates that a report is included in the study.
gsps Contains information about the GSPS to be applied to the images
keyObject Contains information about key objects within this study
other Another type that is unspecified/doesn't yet have special handling. Should be retrievable via a WADO2 URL but that may not provide much user information.
